Lead Generator

Find businesses in any city & category, then flag the ones that need a better web presence.

Python FastAPI Async CSV / Excel export

Quick start — how to run it

If the project is already set up on this machine (virtual environment created and dependencies installed), running it takes three commands:

# 1. go to the project folder
cd lead-generator

# 2. activate the virtual environment
#    Windows (PowerShell):
.venv\Scripts\Activate.ps1
#    macOS / Linux:
source .venv/bin/activate

# 3. start the server
uvicorn app.main:app --reload

Then open http://127.0.0.1:8000, enter a City and Category, and click Find leads. Press Ctrl+C in the terminal to stop the server.

Want real data instead of the sample restaurants? Set LEAD_SOURCE=osm in your .env file for free OpenStreetMap data (any city, any category) — then start the server as above. Or set it just for the current terminal session: 
# Windows (PowerShell)
$env:LEAD_SOURCE="osm"; uvicorn app.main:app --reload

# macOS / Linux
LEAD_SOURCE=osm uvicorn app.main:app --reload
First time on this machine? You need to create the virtual environment and install dependencies first — follow Setup (step by step) once, then use the three commands above every time after.

Overview

Lead Generator is a small web application for sales/marketing prospecting. You enter a city and a category (for example Bangalore / Restaurants, or Pune / Dentists). The app then:

  1. Finds businesses matching that city and category.
  2. Checks each business's website — does it exist, does it load, is it mobile-friendly?
  3. Flags leads — businesses with no website, a broken website, or a non–mobile-friendly website are the prospects worth approaching.
  4. Delivers the list in an on-screen filterable table, downloadable as CSV or Excel.

The business data source is pluggable: the app ships with a zero-config sample source, a free real-data source (OpenStreetMap), and a stub for Google Places — all behind one interface, so you can swap providers without touching the rest of the app.

Key features

  • Any city, any category — the category box is free text (restaurants, gyms, plumbers, salons, hotels…).
  • Concurrent website checks — all sites are checked in parallel with httpx + asyncio, so a search of 60 businesses takes seconds.
  • Three lead signals — no website, broken website, not mobile-friendly.
  • Filterable results table — view all, leads only, or one signal at a time.
  • One-click export — download the lead list as CSV or Excel.
  • Pluggable data sources — mock / OpenStreetMap / Google Places, selected with one env var.
  • Resilient — Overpass endpoint failover, browser-like user agent, graceful handling of timeouts, DNS/SSL errors, and HTTP error codes.
  • Tested — unit tests for the website classifier, the pipeline, and the OSM source (HTTP mocked, no network needed).

How it works

A search flows through four stages. Each stage depends only on the previous stage's output, so any piece can be swapped independently.

City + Category  (from the web form)
▼
BusinessSource.search()  → list of businesses  # mock | osm | google
▼
check_website()  (async, concurrent)  → none | broken | not_mobile_friendly | ok
▼
pipeline.run()  → Lead objects, leads sorted first
▼
Web table  +  CSV / Excel export

The website checker fetches each page (following redirects, ~10 s timeout, browser-like User-Agent) and parses the HTML for a responsive viewport meta tag. The whole pipeline is in app/pipeline.py; the checker is in app/checks/website.py.

Lead classification

Every business is placed into exactly one of four states. The first three are flagged as leads.

StatusLead?How it's detected
No website✅ YesThe data source returned no website for the business.
Broken website✅ YesThe request failed (connection / DNS / SSL error or timeout), or returned an HTTP status of 400 or higher.
Not mobile-friendly✅ YesThe page loads (HTTP 200) but has no <meta name="viewport" content="…width=device-width…"> tag.
OK—Loads successfully and declares a responsive viewport.
Note on mobile-friendliness Detection uses a heuristic (presence of a responsive viewport meta tag), because Google's old Mobile-Friendly Test API is deprecated. For a true mobile score you can plug the PageSpeed Insights API into app/checks/website.py.

Data sources

Choose the source with the LEAD_SOURCE environment variable. All three implement the same BusinessSource interface.

Source (LEAD_SOURCE)Cost / setupCoverageTrustworthy "no website"?
mock (default) None — no keys ~16 sample Bangalore restaurants Demo only
osm — OpenStreetMap Free, no API key Real data, any city & category ⚠️ Sparse — treat as "unknown, verify"
google — Google Places API key + billing (has free credit) Most complete ✅ Authoritative
OpenStreetMap caveat OSM's website tag is often empty, so an OSM "no website" result means "OSM doesn't know of one", not a guaranteed absence. Use these as leads to verify. The broken and not mobile-friendly results are reliable for any business that does list a site.

mock works out of the box. osm just needs LEAD_SOURCE=osm. google requires finishing the stub in app/sources/google_places.py and supplying a key.

Category support

The app is category-agnostic. For the OpenStreetMap source, the free-text category is mapped to OSM tags (in app/sources/overpass.py): about 40 common verticals are mapped directly, and anything else falls back to a fuzzy match so arbitrary categories still return results.

Examples of mapped categories:

restaurants → amenity=restaurant      dentists  → amenity=dentist
cafes       → amenity=cafe            gyms      → leisure=fitness_centre
hotels      → tourism=hotel           pharmacy  → amenity=pharmacy
plumbers    → craft=plumber           salons    → shop=hairdresser / beauty
lawyers     → office=lawyer           supermarket → shop=supermarket

For the Google Places source, the category is used directly as free text ("{category} in {city}"), so no mapping is needed.

Prerequisites

  • Python 3.10 or newer (developed and tested on 3.12). Check with python --version.
  • pip (ships with Python).
  • Internet access — required for the osm/google sources and for checking real websites. The mock source needs the network only to check the sample sites.
  • About 300 MB of disk for the virtual environment (pandas / numpy are the largest dependencies).

Setup (step by step)

These steps set the project up on a fresh machine. Commands are given for both Windows (PowerShell) and macOS/Linux.

  1. Get the project onto the machine

    Copy the entire lead-generator folder to the target machine, or clone it if it lives in a Git repository:

    # If it's in a git repo:
git clone <your-repo-url> lead-generator

  2. Open a terminal in the project folder

    cd lead-generator

  3. Create a virtual environment

    This keeps the project's dependencies isolated from the system Python.

    python -m venv .venv

  4. Activate the virtual environment

    # Windows (PowerShell)
.venv\Scripts\Activate.ps1

# Windows (Command Prompt)
.venv\Scripts\activate.bat

# macOS / Linux
source .venv/bin/activate
    PowerShell blocked the script? If you see "running scripts is disabled on this system", allow it for the current session: 
    Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass

  5. Install dependencies

    pip install -r requirements.txt

  6. Create the configuration file

    Copy the example env file and (optionally) edit it to pick a data source.

    # Windows
copy .env.example .env

# macOS / Linux
cp .env.example .env

    To use real, free OpenStreetMap data instead of the sample data, set LEAD_SOURCE=osm in .env. See Configuration.

  7. Start the app

    uvicorn app.main:app --reload

    Then open http://127.0.0.1:8000 in your browser.

Running the app

# default port 8000
uvicorn app.main:app --reload

# choose a different port if 8000 is taken
uvicorn app.main:app --reload --port 8001

# make it reachable from other devices on your network
uvicorn app.main:app --host 0.0.0.0 --port 8000

--reload auto-restarts the server when you edit code — handy during development, omit it in production.

Using it

  1. Open the app in your browser.
  2. Enter a City (e.g. Bangalore) and a Category (e.g. Restaurants, Gyms, Dentists).
  3. Click Find leads. Results appear in a table with leads highlighted and sorted to the top.
  4. Use the filter buttons (All, Leads only, No website, Broken, Not mobile-friendly) to narrow the view.
  5. Click Download CSV or Download Excel to save the list.

Running tests

With the virtual environment activated:

pytest

The suite mocks all HTTP traffic (using respx), so it runs fast and needs no network. It covers the website classifier (no-URL, responsive, desktop-only, 5xx, connection error, timeout), the pipeline's flagging and lead-first sorting, and the OpenStreetMap source's category mapping and response parsing.

Troubleshooting

SymptomFix
PowerShell: "running scripts is disabled"Run Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass, then activate again.
Port 8000 already in useStart with --port 8001 (or any free port).
OpenStreetMap search errors / times outThe public Overpass servers are busy. The app automatically retries a mirror; just run the search again. You can also set OVERPASS_URL to another mirror.
A real website shows as "Broken" with HTTP 403Some sites block automated requests. This is recorded honestly with the reason; the business is still a valid lead to review manually.
Many OSM results show "No website"Expected — OSM rarely stores the website tag. Verify these manually, or switch to the Google Places source for authoritative data.
uvicorn not foundThe virtual environment isn't active, or dependencies aren't installed. Re-activate and run pip install -r requirements.txt.

Extending — add a new data source

Every source implements one method. To add a provider (SerpApi, Foursquare, a CRM export, etc.):

  1. Create app/sources/yoursource.py with a class that subclasses BusinessSource and implements async def search(self, req) -> list[Business].
  2. Register it in the get_source() factory in app/sources/base.py.
  3. Select it with LEAD_SOURCE=yourname.

Nothing else changes — the pipeline, website checks, web UI, and export are all source-agnostic.

from app.models import Business, SearchRequest
from app.sources.base import BusinessSource

class MySource(BusinessSource):
    name = "mysource"

    async def search(self, req: SearchRequest) -> list[Business]:
        # call your API, build Business objects…
        return [Business(name="…", website="…", address="…")]

Tech stack

  • FastAPI + Uvicorn — async web framework and server.
  • httpx — async HTTP client for website checks and API calls.
  • BeautifulSoup — HTML parsing for the mobile-friendliness check.
  • Jinja2 — server-rendered results page (with a little vanilla JS for filtering).
  • pandas + openpyxl — CSV and Excel export.
  • Pydantic + pydantic-settings — data models and configuration.
  • pytest + respx — tests with mocked HTTP.
